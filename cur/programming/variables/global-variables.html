Manchmal reicht es nicht, sich den Werte einer Variable innerhalb eines Skripts zu merken. Statt dessen brauchen wir die Variable überall in unserem Projekt. Das klassische Beispiel dafür ist der Spielstand; auch wenn das Skript nicht läuft, sollte der Spielstand gespeichert bleiben. Ein anderer Anwendungsfall für eine globale Variable wäre ein Wert, auf den verschiedene Sprites zugreifen sollen, die in einem einzigen Programm miteinander interagieren (vielleicht brauchen alle Deine Sprites den aktuellen Spielstand, zum Beispiel).

In frühen Programmiersprachen waren alle Variablen global. Das führte zu vielen Fehlern, da verschiedene Teile des Programms den selben Namen für verschieden Zwecke verwendeten und sich gegenseitig die Werte überschrieben. Sei daher sparsam mit globalen Variablen; überlege, ob eine Skriptvariable oder ein for Block (der eine Variable nur für diesen Block lokal macht) funktionieren würde.

Um eine Variable global zu machen, musst Du zur Variables Palette gehen und auf den Make a variable Knopf (kein Block -- Du kannst ihn nicht in ein Skript packen!) klicken:

Du wirst aufgefordert, Deiner Variablen einen Namen zu geben. Nachdem Du das gemacht hast, siehst Du ein oranges Variablen-Oval in der Variablen-Palette:

Dieses kann genauso in Skripten gezogen werden, wie die orangen Variablen-Ovale im for Block und dem script variables Block.

Die Checkbox links vom Variablen-Block bestimmt, ob der Wert der Variablen auf der Bühne angezeigt wird. Die Variable watcher (so heißt dieses Display) kann beim Debuggen nützlich sein. Es kann auch permanent an sein, so dass der Benutzer Deines Projekts den Spielstand sehen kann, oder was auch immer Du in der Variablen hast. Wenn Du den Watcher rechtsklickst, kannst Du sein Format ändern.

Wenn Du eine Variable global machst, bekommst Du auch einen "Delete a variable" Knopf zum Löschen einer Variablen.
