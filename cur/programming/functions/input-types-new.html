Der max Block verhält sich unerwartet. Er soll nur für Zahlen funktionieren. Trotzdem kannst Du Text eingeben!




Um das zu verbessern, können wir den Eingabetyp für unseren Block festlegen. Wir begrenzen jetzt den max Block auf Zahlen als Eingabe.

  1. Öffne den Block Editor für den max Block und klicke auf die Eingabe x.



  2. Klicke anschließend den rechten Pfeil in der obigen Popup-Box. Das öffnet die Dialogbox unten. Dort können wir die Form der Aussparung festlegen. Wir wollen eine Aussparung nur für Zahlen (wie unten ausgewählt). Wir können auch festlegen, dass wir der Variable einen Ausgangswert zuweisen; das erinnert etwa an den Block move, der stets mit dem Ausgangswert 10 startet.




  3. Modifiziere sowohl die x als auch die y Variable, so dass sie nur Zahlen als Werte annehmen können und die Ausgangswerte 5 und 10 haben. Dein Block sollte dann so aussehen, wobei die Ausgangswerte im Header sichtbar sind.




  4. Wenn Du OK klickst, solltest Du Deinen Block im Operators Reiter sehen, samt Ausgangswerten. Bemerke auch, dass Du keinen Text mehr eingeben kannst.



Merke: Selbst wenn wir den Eingabetypen für einen Block spezifizieren, kann er immer noch einen ungeeigneten Wert empfangen, wenn eine entsprechende Variable als Argument übergeben wird. (snap überprüft nicht ständig ob der Variablen-Typ mit dem Eingabetypen des Blocks übereinstimmt). Eingabetypen sollten daher nur als Organisations-Werkzeuge verwendet werden, um Dich daran zu erinnern, was die Domäne (Eingabewerte) Deines Blocks sein sollte.
