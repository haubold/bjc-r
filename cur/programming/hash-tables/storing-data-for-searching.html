Das heutige Ziel ist es, eine einfache Suchmaschine zu bauen, die Wörter in einer Liste von Zitaten finden kann. Wir werden unsere Suchmaschine so ähnlich aufbauen wie Google (das besprechen wir später noch ausführlicher). Wie du dir bestimmt denken kannst, haben Suchmaschinen die Anforderung, Dinge schnell zu finden, aber die bisher gelernten Techniken geben uns nicht die Möglichkeit hazu. Wir brauchen etwas schnelles, etwas blitzschnelles.
Hinweis: das Wort Index hat unten verwendet eine andere Bedeutung als der Suchmaschinen-Index, der im Rest dieses Kapitels benutzt wird.

Wie du weißt, erlauben einem Listen, mehrere Stücke Daten in einer Variable aufzubewahren. Du kannst sie schnell wieder aufrufen, indem du den Index der Daten, die du finden möchtest, angibst. Der Index ist die Nummer in einer Liste. Es gibt eine sehr ähnliche Datenstruktur, die Hashtabelle genannt wird, die sehr ähnlich funktioniert, aber stattdessen auch ein Wort oder eine Phrase als Index erlaubt. Du kannst den Unterschied im Bild auf der rechten Seite sehen. Statt einer Nummer wird der Name der Person als Index verwendet.

Dies ist eine unglaublich mächtige Art und Weise, Daten aufzubewahren, die sich auf andere Daten beziehen. Zum Beispiel könnte man ein Telefonbuch mit einer Hashtabelle implementieren, die den Namen der Person als Index und die Telefonnummer als Daten verwendet. Wenn du später die Nummer von jemandem herausfinden möchtest, gibst du einfach den Namen an und nimmst die damit verbundene Telefonnummer aus der Hashtabelle. Dieser Kurs stellt dir drei sehr schnelle Blöcke zur Verfügung, um mit Hashtabellen zu arbeiten.

Der erste Block erlaubt es dir, eine Liste in eine Hashtabelle umzuwandeln. Der erste Parameter ist eine leere Liste und der zweite die Größe der Hashtabelle (die Anzahl der Elemente, die hinein passen sollen). Wenn dieser Block ausgeführt wurde, ist die übergebene Liste eine Hashtabelle!

Hinweis: Es ist in der Regel besser ungefähr doppelt so viel Platz in der Hashtabelle zu haben, als du wirklich bräuchtest. Auch, aus technischen Gründen, funktionieren Hashtabellen am besten, wenn ihre Größe eine Primzahl ist. Du kannst sie entweder zu einer verhältnismäßig großen Zahl setzer oder einen anderen Weg benutzen, der dir einfällt. Die Primzahl sollte zu der Menge von Daten passen, die aufbewahrt werden soll -- 53 mag für kleinere Datensätze genügen, größere Projekte (so wie das letzte Projekt dieses Kapitels) sollten eher eine Größe von 521 haben.

Dieser Block erlaubt dir, neue Daten in die Hashtabelle einzutragen. Der erste Parameter gibt dem Eintrag einen Namen, wie zum Beispiel "Max Mustermann" in einem Telefonbuch. Der zweite Parameter ist die mit dem ersten Block generierte Hashtabelle. Der dritte Parameter ist der Wert (eine Nummer, Phrase, Liste, was auch immer), auf den sich "Max Mustermann" bezieht. Dieser Block wird die angegebene Information in die Hashtabelle eintragen, sodass sie später wieder aufgerufen werden kann.

Nachdem du die Information eingetragen hast, wird dieser Block sie dir wieder schnell herausholen. Er gibt den Wert (z.B. die Telefonnummer) für den Schlüssel (z.B. "Max Mustermann"), den du angibst. Du musst auch die mit dem ersten Block generierte Hashtabelle angeben.

[Optional] Information for the Curious

Hash tables are a tremendous tool for a computer scientist's toolbox because they allow for very quick information storage and retrieval. Hash tables make use of two regular lists: one to store the index data, or keys, and the other to store the data, or values. Each key is paired with its own value and entered at equal points in the two lists (see red highlighted region of image).

When someone wants to know Cleopatra's phone number, it is then possible to move through the list of keys until Cleopatra's name is discovered, then look up the phone number from the list of values with the equivalent index (if Cleopatra's name is stored in slot 4 of the key list, her phone number will be stored in slot 4 of the values list). In reality, it is not necessary to move through the list in order; there is a special technique called hashing (hence the name 'hash tables') that makes it possible to find elements in far less time. We won't go into the details of that technique here, but the combination of hashing with the two lists is what gives the hash table its power.

Feel free to check out the Wikipedia article on hash tables for more information.
